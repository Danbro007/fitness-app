## 已修复缺陷

### P1：创建计划后看起来“按钮没反应”

复现条件：在周日创建默认日期为次日的计划。SQLite 已成功写入，但页面只展示本周，次日属于下周，保存后界面没有任何可见变化。另一个显示缺口是同一天有多条计划时只展示第一条。

修复：创建成功后立即导航到新计划详情；本周同一天存在多条计划时全部展示。集成测试现在验证“点击确认 → 数据库出现 → 自动进入对应详情 → 编辑 → 重建后仍存在”。

### P1：日历标题范围与实际可达日期不一致

复现：周日创建次日的 PlanA 和 PlanB。标题显示“7 月 6—14 日”，但界面实际只渲染 6—12 日；且“已计划”按日期去重，两个计划误显示为 1。

修复：周视图与标题一致覆盖 9 天，计数改为计划条数，同日摘要和日期详情展示全部计划。真实点击已验证两条都可达。

### P2：非法计划日期泄漏英文框架异常

复现：输入 `2026-99-99` 并创建计划，页面直接显示 `DateTimeParseException` 英文文本。

修复：将日期解析错误转换为“请输入有效日期（格式 YYYY-MM-DD）”，并使弹出表单可滚动，避免小屏或错误文本挤出操作按钮。

### P1：首页动作库返回后其他快速入口响应

新增连续设备测试，严格执行“点击动作库 → 返回首页 → 点击记录饮食 → 饮食 Tab 选中”。该链路修复后持续可用，并纳入全量回归，防止导航状态残留导致后续点击失效。

### P1：Android 8/8.1 GIF 解码兼容性

应用最低支持 API 26，但 GIF ImageDecoder 工厂仅支持 API 28+，导致 Lint 阻断并存在旧系统运行风险。

修复：直接使用系统版本门禁；API 28+ 使用 `ImageDecoderDecoder`，API 26–27 使用 `GifDecoder`。修复后 Lint 无错误。

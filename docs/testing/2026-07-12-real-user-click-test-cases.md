# i Fitness 真实用户点击测试用例

- 版本：1.0
- 日期：2026-07-12
- 测试对象：原生 Android App（不是 HTML 原型）
- 设备基线：Pixel 8 Pro AVD，Android 16 / API 36
- 操作原则：只通过屏幕上用户可见的控件点击、输入、滑动和系统返回完成业务流程
- 数据原则：测试开始前清空 App 数据；AI 结果必须确认后才允许进入正式业务数据

## 1. 证据规则

每条用例必须记录以下结果，不能只填写“通过”：

1. 点击前截图、关键结果截图。
2. 实际看到的页面标题、按钮状态、错误提示或成功反馈。
3. 涉及写入的流程，检查 SQLite 中对应记录是否新增或更新。
4. 每组用例完成后检查 logcat 中是否出现崩溃、ANR 或未捕获异常。
5. 失败时记录首次出现问题的最短点击序列；修复后从该序列的第一个入口重新点击。

状态使用：`未执行 / 通过 / 失败 / 阻塞 / 不适用`。

## 2. 首次启动与本地档案

### CLICK-ONBOARD-001 干净安装首次启动

- 前置：清空 `com.shanqijie.fitnessapp` 数据后启动。
- 步骤：等待启动完成，不点击任何控件。
- 预期：显示“先完成训练设置”；不应短暂进入首页后再跳回；无崩溃或无限加载。
- 证据：首次设置页截图、启动 logcat。

### CLICK-ONBOARD-002 必填项为空时提交

- 步骤：滑动到页面底部，直接点击保存/完成按钮。
- 预期：停留在当前页；明确指出必填项，不写入已完成 onboarding 状态。
- 证据：错误提示截图、preferences 表状态。

### CLICK-ONBOARD-003 数值边界与非法输入

- 步骤：依次输入非法出生年、身高、体重、每周训练天数和训练时长，再点击保存。
- 预期：输入不能造成崩溃；每个非法字段有可理解反馈；合法值不会被错误清空。
- 证据：每类错误提示、logcat。

### CLICK-ONBOARD-004 完成首次设置

- 步骤：输入昵称、出生年、身高、体重，选择目标，填写每周频率和时长，点击完成。
- 预期：进入首页；底部只有“首页、计划、训练、饮食、我的”五项；重启后不再出现首次设置。
- 证据：首页截图、user profile 与 onboarding preference、重启截图。

## 3. 首页、快速入口与连续导航

### CLICK-HOME-001 首页核心信息

- 步骤：进入首页并从顶部滑到底部。
- 预期：页面内容可完整滚动；主训练操作只有一个；周进度和快速入口可见；无内容被底栏遮挡。

### CLICK-HOME-002 快速入口“记录饮食”

- 步骤：点击“记录饮食”。
- 预期：进入饮食页；底栏“饮食”处于选中状态；页面仍可继续点击。

### CLICK-HOME-003 快速入口“动作库”后返回再点其他入口

- 步骤：首页点击“动作库” → 在动作库点击页面返回按钮 → 首页点击“记录饮食” → 点击底栏“首页” → 再次点击“动作库”。
- 预期：每次点击只导航一次；返回后“记录饮食”和第二次“动作库”都响应；没有透明遮罩拦截点击。
- 重点：这是用户报告问题的首要回归用例。
- 证据：每一步截图、点击后的页面标题、logcat。

### CLICK-HOME-004 动作库后使用系统返回

- 步骤：首页点击“动作库” → 使用 Android 系统返回 → 点击“记录饮食”。
- 预期：系统返回与页面返回行为一致；后续点击仍有效。

### CLICK-HOME-005 五个一级导航循环

- 步骤：依次点击“首页 → 计划 → 训练 → 饮食 → 我的 → 首页”，连续执行两轮。
- 预期：选中态和页面一致；无一次点击无响应；无错误保留上个页面内容。

### CLICK-HOME-006 快速连续点击防重复导航

- 步骤：快速双击“动作库”，返回；快速双击“记录饮食”。
- 预期：不创建重复页面、不闪退、不出现无法返回或点击失效。

## 4. 计划与训练日历

### CLICK-PLAN-001 新建次日计划

- 步骤：计划页点击新增 → 输入名称 → 保持默认次日 → 点击确认创建。
- 预期：SQLite 新增一条计划；立即进入该计划详情，不能出现“按钮没反应”的静默状态。

### CLICK-PLAN-002 同一天创建两条计划

- 步骤：连续创建两条相同日期、不同名称的计划 → 返回计划页。
- 预期：当天两条计划均可见、均可点击，名称和详情对应正确。

### CLICK-PLAN-003 非法日期和空名称

- 步骤：输入非法日期、空名称并分别点击确认。
- 预期：不崩溃；提示日期或名称问题；编辑内容保留以便修改。

### CLICK-PLAN-004 编辑计划并重启

- 步骤：进入计划详情 → 编辑名称和日期 → 保存 → 强制停止并重启 App。
- 预期：修改立即显示；重启后仍存在；旧名称不残留。

### CLICK-PLAN-005 日历周/月/年切换

- 步骤：依次点击周、月、年；切换上月/下月；点击有计划和无计划日期。
- 预期：视图和日期正确变化；空日期显示休息提示；有计划日期可进入详情。

### CLICK-PLAN-006 AI 四周计划确认边界

- 步骤：生成计划草稿 → 未确认时返回并检查计划数 → 再进入并确认。
- 预期：未确认不创建正式计划；确认一次创建四周计划；重复点击不重复创建。

## 5. 动作库

### CLICK-LIB-001 打开、筛选和清除筛选

- 步骤：打开动作库 → 依次点击不同身体部位筛选 → 再切回全部。
- 预期：每个筛选点击立即改变列表；不会出现点击第一个筛选后其他筛选失效。

### CLICK-LIB-002 搜索、清空和再次搜索

- 步骤：搜索“高位下拉” → 打开结果 → 返回 → 清空 → 搜索另一个动作。
- 预期：结果、详情和返回上下文正确；输入框与列表始终可继续操作。

### CLICK-LIB-003 首页来源的动作详情

- 步骤：首页进入动作库 → 打开动作详情 → 点击“添加到计划”。
- 预期：没有可添加计划时给明确提示，不能静默无响应或崩溃；有当前计划时写入正确计划。

### CLICK-LIB-004 从计划编辑进入动作库

- 步骤：计划详情 → 编辑 → 从动作库添加 → 搜索 → 打开详情 → 添加。
- 预期：动作添加到原计划；返回计划编辑页；重启后仍存在。

### CLICK-LIB-005 从进行中训练进入动作库

- 步骤：训练中选择添加动作 → 动作库搜索并添加。
- 预期：只添加到当前训练，不污染原计划；返回训练页且训练状态不丢失。

## 6. 训练完整流程

### CLICK-WORKOUT-001 从首页开始训练

- 步骤：首页点击主训练按钮 → 训练准备 → 开始训练。
- 预期：创建且仅创建一条进行中 session；进入沉浸训练页；全局底栏隐藏。

### CLICK-WORKOUT-002 记录一组与防重复点击

- 步骤：修改重量/次数/感受 → 快速双击完成一组。
- 预期：只新增一组记录；进入休息状态；组数增加一次。

### CLICK-WORKOUT-003 休息计时和跳过

- 步骤：完成一组 → 等待数秒 → 点击跳过休息。
- 预期：倒计时递减；跳过后回到训练；按钮不会残留不可点击遮罩。

### CLICK-WORKOUT-004 中途退出和恢复

- 步骤：记录至少一组 → 系统返回/切到其他 App → 强制停止并重启。
- 预期：App 提供继续训练；已完成组与当前动作不丢失；不会新建第二条 session。

### CLICK-WORKOUT-005 完成训练

- 步骤：点击结束训练 → 取消一次 → 再次结束并确认。
- 预期：取消保持训练中；确认后 session 完成并进入总结；首页周进度更新。

## 7. 饮食

### CLICK-FOOD-001 手动记录非法输入

- 步骤：打开新增餐食 → 空名称、负数、非数字分别提交。
- 预期：明确提示；表单内容保留；不写入 SQLite。

### CLICK-FOOD-002 手动记录并重启

- 步骤：输入食物名、热量、蛋白质、碳水、脂肪 → 保存 → 重启。
- 预期：今日记录和汇总立即更新；重启后仍存在且不重复。

### CLICK-FOOD-003 图片估算未配置 AI

- 步骤：进入图片识别入口，在未配置服务商时继续。
- 预期：明确引导配置或说明不可用；不能无限加载或静默无响应。

### CLICK-FOOD-004 图片估算确认边界

- 步骤：生成估算草稿 → 返回 → 再进入 → 明确确认。
- 预期：确认前不写正式 food log；确认后只写一条；重复确认不重复。

## 8. 我的、设置与本地数据

### CLICK-PROFILE-001 编辑档案和体测

- 步骤：我的 → 编辑 → 修改昵称、目标和体测 → 保存。
- 预期：我的页立即显示新数据；重启后保留。

### CLICK-PROFILE-002 场地与器械

- 步骤：进入场地设置 → 新增场地 → 设为默认 → 连续切换多个器械 → 返回再进入。
- 预期：每个开关响应；默认场地和器械选择持久化；不会点击一个后其他开关失效。

### CLICK-PROFILE-003 AI 服务商联动

- 步骤：切换 OpenAI、Gemini、千问，观察 endpoint/model → 输入明显无效密钥并测试连接。
- 预期：服务商切换更新对应字段；失败有明确反馈；密钥不以明文出现在页面或日志。

### CLICK-PROFILE-004 备份导出取消

- 步骤：点击导出 → 在系统文件选择器取消 → 返回 App。
- 预期：App 可继续操作；不显示虚假成功。

### CLICK-PROFILE-005 无效备份导入

- 步骤：选择格式错误的备份文件。
- 预期：明确失败；原有档案、计划、训练和饮食数据不变。

### CLICK-PROFILE-006 重置数据确认

- 步骤：点击重置 → 取消 → 再次重置并确认。
- 预期：取消不改变数据；确认后清除个人业务数据并返回首次设置/干净状态。

## 9. 生命周期、布局与稳定性

### CLICK-LIFE-001 每个二级页系统返回

- 步骤：动作库、动作详情、计划详情、计划编辑、场地、智能设置、备份、关于逐页使用系统返回。
- 预期：回到正确来源页；不会直接退出或进入错误一级 Tab。

### CLICK-LIFE-002 前后台与 Activity 重建

- 步骤：在首页、表单、动作库搜索、训练中分别切后台再回来；旋转设备一次。
- 预期：无崩溃；已提交数据保留；未提交表单按产品约定保留或明确重置。

### CLICK-LIFE-003 长文本、键盘和小屏可达性

- 步骤：输入长昵称/计划名；打开键盘；滚动到底部；检查关键按钮。
- 预期：文本不挤出屏幕；键盘不会永久遮挡保存按钮；可通过滚动到达。

### CLICK-LIFE-004 高频点击稳定性

- 步骤：在非破坏性按钮上进行 20 次快速切换，包括一级导航、动作筛选、日历模式。
- 预期：无 ANR、崩溃、重复写入或最终不可点击状态。

## 10. 退出标准

只有同时满足以下条件才可结束测试：

1. 所有 P0/P1 用例均为通过；不存在“偶发点击无响应”。
2. 所有失败都已记录最短复现步骤、完成修复并从入口重新点击通过。
3. 涉及数据的关键流程已用 SQLite 证明写入次数和目标对象正确。
4. 完整点击回归期间无 App 崩溃、ANR 或未捕获异常。
5. 无密钥导致的真实 AI 联网用例必须标记为阻塞/未覆盖，不能冒充通过。

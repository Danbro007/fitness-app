# ADR-0001：伤病动作过滤采用保守策略

- 状态：已接受
- 日期：2026-07-22

## 背景

训练计划会读取用户以自由文本填写的伤病和注意事项，并自动判断需要避开的动作。自由文本可能不完整、含糊或无法可靠映射到动作；错误地保留高风险动作可能给用户带来伤害。

## 决策

伤病约束优先于训练目标、历史表现和渐进加重规则。

系统判断某个动作与已记录伤病相关时，必须从计划草稿中排除该动作，不能只降低重量后继续安排。系统无法确定动作是否安全时，也采用保守策略：排除疑似相关动作，并在计划草稿中向用户说明排除原因。

该判断属于训练计划安全过滤，不构成医学诊断。

## 影响

- 计划生成链路需要在动作进入草稿前完成伤病过滤。
- 本地兜底计划与联网 AI 计划必须遵守相同的伤病过滤结果。
- 计划草稿需要展示被排除动作及原因，便于用户理解计划变化。
- 保守过滤可能产生误排除，但不得以提高动作覆盖率为由绕过安全约束。
- 用户可以主动把被排除动作恢复到计划草稿中；系统不得静默恢复。
- 用户确认恢复后形成持久化的伤病过滤例外，后续生成计划时继续生效。
- 用户修改伤病/注意事项后，已有过滤例外自动失效，不能继续沿用旧确认。
- 恢复动作前必须展示排除原因和风险提示，并要求用户明确二次确认。
- 用户因“身体不适”提前结束训练后，下一周计划生成前必须要求用户复核伤病/注意事项；复核完成前阻止生成。复核可以修改信息，也可以明确确认现有信息无需修改。
